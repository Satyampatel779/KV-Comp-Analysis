# Screenshots — capture checklist

Drop PNGs here with the **exact names below** (they're referenced by the main `README.md`).
Until you add them, GitHub shows a broken-image icon — that's expected.

**Tips for clean shots**
- Run a real subject first so the screen is populated. `120 DEERCREST CL SE` is a good demo subject.
- Turn the **KV comp criteria** toggle ON in the sidebar.
- Use a maximised browser window; crop to the relevant panel (no bookmarks bar / desktop clutter).
- Light theme tends to read better in a README; keep it consistent across shots.
- On Windows: `Win + Shift + S` to snip. Save straight into this folder.

| File name | What to capture |
|---|---|
| `hero.png` | The money shot — top of the app with the **subject card + ranked comps table + map** visible (scroll so a bit of each shows). This is the first thing reviewers see. |
| `ranked-comps.png` | The **Ranked comparable sales** table: the `#`, KV ✓, Sale $, today-adjusted, $/land-sqm, distance, recency, and **score** columns. |
| `score-breakdown.png` | The **"Score breakdown — how each comp's score is built"** expander opened, showing the per-factor points. |
| `value-band.png` | The four metrics row (**Comps used / Median / Today-adjusted median / Confidence**) plus the blue **implied value band** line. |
| `map.png` | The **pydeck map** — blue subject pin, comps coloured by recency, lines from comps to the subject. |
| `assistant.png` | The **Ask the assistant** panel after asking "What is a fair offer range?" — answer with `[#n]` citations + the disclaimer. |
| `subject-imagery.png` | The **subject card** with the pinned aerial photo (and Street View if you set a Google key). |

Optional extras you can add and reference if you like: `pdf-handoff.png` (the exported PDF),
`manual-subject.png` (the off-market entry form), `charts.png` (price/distance/timeline tabs).
